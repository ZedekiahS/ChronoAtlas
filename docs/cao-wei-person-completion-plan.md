# 曹魏人物补全计划

目标不是只堆人名，而是把《三国志》魏书与《资治通鉴》中曹魏相关人物整理成可审核、可追踪、可扩展的人物资料库。

## 当前状态

已完成《三国志·魏书》目录级人物基础导入：

- 覆盖魏书本纪、后妃传、汉末群雄传、曹魏宗室传、文臣武将传、方技传。
- `data/china-persons.json` 中已有 165 个魏书目录人物入口。
- 新增人物均有至少一条《三国志·魏书》来源引用，以及一条“待逐段整理”的人物年表 seed。
- 魏书三十《乌丸鲜卑东夷传》属于族群、边疆和地理政治材料，已作为来源追踪，不进入单个人物表。

这一步是“全量建档骨架”，不是逐传精读完成。新增 `stub` 人物后续还需要补：

- 生卒与字号核对。
- 本传逐段生平节点。
- 裴松之注异文与补充人物。
- 《资治通鉴》编年互校。
- 人物关系、事件回链和原文摘录。

## 补完标准

一个人物进入“可用档案”至少需要：

- `china-persons.json` 中有稳定 `id`、姓名、字号、生卒或生卒未详、主要政权、角色和摘要。
- 至少有一条《三国志》或《资治通鉴》来源；有本传者优先挂本传。
- 核心人物有 4 条以上生平节点，边缘人物至少保留来源化提及或待补状态。
- 参与重大事件的人物回链到 `events-180-280.sample.json`。
- 明确的人物关系进入 `china-person-relations.json`。
- 未逐段核对裴松之注和《资治通鉴》前，`coverageStatus` 不标成 `full`。

## 推进顺序

1. 魏书逐传细化：先从曹操、曹丕、曹叡、曹魏宗室、五子良将、淮南三叛、邓艾钟会开始。
2. 《资治通鉴》卷五十八至卷八十一按年互校。
3. 裴松之注中的《魏略》《世语》《九州春秋》等扩展人物与异文。
4. 同名、字号、官称、谥号、爵号逐步整理进别名体系。

## 覆盖追踪

机器可读清单在 `data/cao-wei-person-coverage-plan.json`。运行：

```bash
npm run coverage:cao-wei
```

可以看到当前魏书目录人物、已建档状态、生平节点数量和关系数量。
